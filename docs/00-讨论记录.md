### 2026-03-23 — 开发资源确认 + 竞品调研 + MVP范围讨论

- **本轮确认信息**：
  - 目标用户：三类均需覆盖，但 B 端（培训机构）defer 到 v2.0
  - AI 视频：初步用 AI 生成教学内容，后期考虑授权专业教练内容
  - 开发方式：个人开发，使用 Claude Code + Cursor；预期1个月（工作日晚上+周末，约40–60小时）

- **竞品调研结论**：
  - 国内市场：**完全空白**，台球 App 全为娱乐游戏，无训练工具类产品
  - 国际标杆：**DrillRoom**（AI Pool Trainer）是最接近对标，核心护城河为手机摄像AI实时分析；但无中文、不含斯诺克/中式台球、无训练日志/心得功能
  - 差异化空间：中文+本土球种内容 + 训练日志/心得 + 知识库（理论体系）

- **AI视频策略调整**：
  - 原始想法：用AI生成教学视频 → 风险：技术动作精度要求高，AI视频生成用于专业运动指导效果不稳定，内容错误对训练有害
  - **调整决策**：v1.0 以 AI 辅助生成结构化**图文内容**为主（Claude 生成知识体系+Drill说明）；视频功能 defer 到 v1.1 或 v2.0；AI摄像实时分析（类DrillRoom）列为 v2.0 核心差异化功能
  - 原决策 → 新决策 → 原因：AI视频生成 → AI图文内容 → 运动技术精度要求高，图文可控且足够实用

- **MVP 范围确认（草案，待用户最终确认）**：
  - **P0（v1.0必须）**：训练日志 + 知识库（AI生成图文）
  - **P1（v1.0尽量）**：训练计划模板 + 进度追踪图表
  - **暂缓**：视频内容、AI摄像分析、B端管理、社交功能

- **仍待确认**：
  - 用户是否认同「v1.0纯C端，B端defer」策略？
  - 知识库内容优先覆盖哪个球种？（斯诺克 / 中式台球 / 9球 / 全覆盖）
  - 是否需要用户账号系统，还是v1.0纯本地数据？

### 2026-03-23 — 产品核心重定义：动作库 + 完整信息架构

- **本轮确认信息**：
  - v1.0 只做C端（培训机构是未来目标客户，但使用C端功能）
  - 球种：中式台球 + 9球（斯诺克 defer）

  - Android：暂不确定，CloudKit 方案不影响未来评估
  - 开发速度：无时间限制，Claude Code + Cursor 可高效实现，专注于完整产品定义

- **关键设计决策**：
  - **产品核心参照「训记」App**：台球 Drill 视同健身动作，建立「台球动作库」，是产品的内容基础和核心差异化

  - **教学视频**：纳入产品，视频来源由用户解决；v1.0动作库内容以图文为基础，视频位置预留
  - **训练记录模型**：每次记录 = 球种 + 多个Drill × 多组 × 每组进球数 + 心得
  - **分级训练计划**：参照「新锐台球学院」四级体系（入门→初级→中级→高级）

- **训练内容体系（已调研）**：

  - 参照：新锐台球学院分级体系 + Dr. Dave Pool Info + DrillRoom 分类

### 2026-03-23 — 新增角度感知训练模块，确认交互细节

- **本轮确认信息**：
  - 训练记录进球数：**数字键盘输入**（精确）
  - 官方训练计划：**基于调研结果自行设计**（参照新锐台球学院四级体系）
  - 球台示意图：**动画**（展示母球运动路线，包括击球前路径 + 接触后母球走位 + 目标球路线）

- **新增功能：角度感知训练（独立第5个Tab）**
  - 设计理由：角度瞄准法是最准确的瞄准系统，该模块专门训练玩家的角度视觉感知能力，是市场上所有台球App都没有的独创功能
  - 子功能一：**角度测试**（随机出题，用户目测切入角并数字输入，动画揭示答案+接触点位置）
  - 子功能二：**进球点对照表**（交互式滑块实时更新接触点，完整角度对照表）
  - 核心公式：偏移量 = sin(α) × R，纯本地计算，零网络依赖
  - 信息架构：从4 Tab 升级为5 Tab（训练/动作库/角度/历史/我的）

- **仍待确认**：
  - 角度测试出题策略：纯随机 vs 自适应（根据历史误差重点出薄弱角度）？
  - 难度分级容差：宽松±10° / 标准±5° / 严格±2°？

- **已更新文档**：`docs/04-功能规划.md`（新增F10/F11）、`docs/05-信息架构与交互设计.md`（5 Tab + 角度Tab详细设计）

### 2026-03-23 — 多源训练计划调研整合 + 角度测试设计定稿

- **本轮确认信息**：
  - 角度测试出题：**随机 + 自适应权重**（按误差率加权，角袋和中袋分开统计）
  - 难度容差：**专业级 ±3°**（唯一难度）
  - 训练计划：**多源调研整合**，不仅限新锐台球学院

- **多源调研结论**（6个来源）：
  - **国内**：新锐台球学院六级认证体系 + 伟林培训三阶段 + 王桁教练三步递进
  - **国际**：Billiard University (Dr. Dave) BU考试体系（Exam I/II/IV）+ BIZU Billiards 单次会话结构 + Billiard & Pool Center 四阶段
  - 整合后形成 **L0–L4 五级体系**，与中国六级认证 + BU考试体系双向对应
  - 设计 **6套官方训练计划**（入门8周/基础杆法6周/走位8周/中级10周/高级8周/全能持续）

- **角度测试自适应算法定稿**：
  - 出题权重 = 基础权重(30%) + 误差权重(70%)
  - 以5°为单位分18个区间，各区间独立记录最近10次误差
  - 角袋(0–75°, 60%比例) vs 中袋(15–60°, 40%比例) 分开统计和出题
  - 每次建议20题，历史以5题为组追踪进步趋势

### 2026-03-23 — 技术架构确认

- **确认决策**：
  - 最低 iOS 版本：**iOS 17**
  - Drill 内容存储：**CloudKit 公开数据库**（内容可随时热更新，无需发版）
  - 球台动画：用户在第三轮讨论中主动提出，为 Drill 详情页的路径动画（非游戏，是教学示意）

- **技术栈定稿**：
  - SwiftUI + iOS 17
  - SwiftData（本地离线数据库）
  - CloudKit（私有库：用户数据同步；公开库：Drill 内容 + 官方计划）
  - Sign in with Apple
  - SwiftUI Canvas（球台动画）
  - 纯 Swift 数学（角度感知训练，sin 函数）

- **架构核心原则**：离线优先（球馆无网也可用）+ 无自建服务器 + 内容热更新

- **主要技术风险**：72个Drill的动画路径数据需手工设计（JSON格式），工作量较大

### 2026-03-23 — 登录方式调整：增加微信 + 手机验证码

- **背景**：中国市场用户普遍习惯微信登录，Sign in with Apple 单一方式门槛偏高
- **决策**：
  - 原方案：仅 Sign in with Apple + CloudKit
  - **新方案：微信登录 + 手机验证码 + Sign in with Apple 三种并存**
  - 用户数据层从 CloudKit 私有库改为 **LeanCloud**（原生支持三种登录，数据在中国大陆）
  - CloudKit 公开库保留，继续用于 Drill 内容热更新
- **架构变更原因**：微信 OAuth 和短信验证码均需后端服务，LeanCloud 是无需自建服务器的最简方案
- **匿名使用**：未登录用户可浏览动作库、使用角度训练，训练记录仅本地存储
- **微信开放平台**：需提前申请，审核约 1–3 天，是上线前需完成的前置事项

- **已更新文档**：`docs/06-技术架构.md`（v0.3）、`.kiro/steering/product.md`

### 2026-03-23 — 商业化模式确认：Freemium 详细方案

- **商业模式**：免费增值（Freemium）+ 订阅（月/年）+ 买断（终身），无广告
- **免费内容原则**：以「等级梯度」控制开放比例，入门内容完全免费，高阶内容渐进锁住
  - 动作库：L0全开放，L1约70%，L2约40%，L3约20%，L4约10%，共约40/72项免费
  - 官方计划：前两个（L0→L1入门计划、L1基础杆法专项）完全免费
  - 训练日志录入无限免费；历史查看免费60天
  - 角度训练对照表完全免费；测试免费每天20次
- **定价**：月订阅¥18、年订阅¥88（月均¥7.3）、终身买断¥198
- **关键合规**：
  - 数字内容必须使用 Apple IAP，不可引导外部付费渠道
  - 已开放的免费功能不可降级为付费
  - 微信开放平台需上线前提前申请审核
