# 路线图与 MVP

## 1. MVP 定义

**目标**：交付可用于真实训练场景的 C 端 iOS App，覆盖「动作库 + 训练记录 + 训练计划 + 角度训练」四大核心模块。

**主力用户**：Persona A（普通爱好者）+ Persona B（进阶玩家），共用同一套核心功能。

**发布渠道**：App Store（中国区）。

**MVP 必含功能（P0）**：

| 功能 | 说明 |
|---|---|
| 动作库（Drill Library） | 8大类约72项Drill，含球台动画 + 图文说明 + 教学视频位置预留 |
| 训练记录 | 按计划或自由记录，数字键盘输入每组进球数，含心得 |
| 官方训练计划 | 5级分级，6套官方计划（入门→初学→进阶→综合→高级→专业），按单次会话结构 |
| 角度感知训练 | 角度测试（自适应权重，±3°专业容差）+ 进球点对照表 |
| 账号系统 | Sign in with Apple + CloudKit 数据同步 |
| 历史日历 | 日历视图 + 单次训练详情 |

**尽量包含（P1）**：

| 功能 | 说明 |
|---|---|
| 自定义训练计划 | 用户从动作库自由组合Drill |
| 训练统计图表 | 成功率趋势、训练频率、大类雷达图 |
| Drill收藏 | 收藏常用训练项目 |
| 个人偏好设置 | 球种偏好、训练目标、通知 |

**明确不做（v1.0）**：

- 教学视频内容制作（位置预留，视频由内容方后续填入）
- AI摄像/AR实时分析（v2.0）
- 培训机构B端（v2.0）
- 社交/排行/约球功能
- iPad 专项优化

## 2. 官方训练计划内容（六套，多源调研结果）

基于六个来源整合（新锐台球学院 + 伟林培训 + 王桁教练 + Billiard University + BIZU Billiards + Billiard & Pool Center），设计以下官方计划：

### 五级分级体系

| App级别 | 中国参照 | BU参照 | 能力特征 | 达标标准 |
|---|---|---|---|---|
| L0 入门 | 一级 | Pre-Exam | 动作稳定，能打短距离直线球 | 半台直球连进5颗 |
| L1 初学 | 二级 | BU Bachelors | 基础杆法，中台角度入袋，简单走位 | 五分点各角度15球进6颗以上 |
| L2 进阶 | 三四级 | BU Masters | 稳定走位，加塞入门，偶尔清台 | 五分点连进10颗 |
| L3 熟练 | 四五级 | BU Exam II | 稳定清台，防守意识，多库走位 | 五分点连进15颗 |
| L4 专业 | 五六级 | BU Doctorate | 高级球形，竞技状态，复杂走位 | 远台连进15颗 |

### 六套官方计划概要

| 计划名 | 目标级别 | 周期 | 每周频次 | 单次时长 | 核心主题 |
|---|---|---|---|---|---|
| 新手入门计划 | L0→L1 | 8周 | 3次/周 | 60分钟 | 基础动作 → 角度入门 → 基础杆法 |
| 基础杆法专项 | L1 | 6周 | 3次/周 | 75分钟 | 五种杆法（定/高/低/斯登/加塞）系统掌握 |
| 走位突破计划 | L1→L2 | 8周 | 4次/周 | 75–90分钟 | 不吃库→一库→分离角→连打走位 |
| 中级综合突破 | L2 | 10周 | 4次/周 | 90分钟 | 准度+加塞+特殊球路+蛇彩+实战 |
| 高级专项：加塞与多库 | L3 | 8周 | 4–5次/周 | 90–120分钟 | 加塞精准+多库走位+防守+高级蛇彩 |
| 全能综合（持续） | L3→L4 | 无限 | 5次/周 | 90–120分钟 | 热身+准度+走位专项+综合球形+实战 |

### 单次会话标准结构（参照 BIZU Billiards 最佳实践）

```
每次训练（60–120分钟）
├── 热身（10分钟）：直线球 + 当天主题杆法找手感
├── 专项训练（30–50分钟）：当日计划Drill，记录每组进球数
├── 综合/实战（15–20分钟）：当级别对应的球形/连打/Ghost Game
└── 复盘记录（5分钟）：完成数据录入 + 心得备注
```

## 5. 反馈与迭代机制

- **TestFlight内测**：邀请5–10名台球爱好者，覆盖入门和进阶两类用户
- **上线后监测**：App Store 评分/评论 + 用户反馈入口
- **数据驱动迭代**：观察哪类Drill被最多添加（动作库热度）→ 优先补充对应视频内容
- **v1.1迭代焦点**：根据实际使用情况，优先在「训练记录」或「内容深度」方向加强

## 6. 暂缓项与原因

| 暂缓功能 | 暂缓原因 |
|---|---|
| 教学视频内容 | v1.0位置预留，视频来源由内容方解决后填入；不阻塞App发布 |
| AI摄像实时分析 | 技术复杂度高（CV+AR），是v2.0核心护城河 |
| B端培训机构管理 | 需要多角色/多用户逻辑，开发量大 |
| 社交功能 | 与「专注训练」定位有冲突 |
| 斯诺克内容 | v1.0先聚焦中式台球+9球 |
| iPad专项优化 | v1.0 iPhone优先，iPad兼容性做但不专项适配 |
