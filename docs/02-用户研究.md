# 用户研究

## 1. 研究目标

明确三类目标用户的核心使用场景、需求深度与优先级，支撑功能规划中的 P0/P1 判断。

## 2. 用户分群与 Persona

> ⚠️ 以下 Persona 为**假设**，待用户访谈或问卷验证。

### 2.1 Persona A — 业余台球爱好者「小李」

- **年龄**：25–40岁
- **台球经历**：打球2–5年，每周1–3次，以中式台球/9球为主
- **学习方式**：看抖音/B站教学视频，偶尔找朋友指点
- **核心痛点**：视频内容碎片化，看完就忘，不知道自己该练什么；练球没有计划，感觉进步很慢
- **期望**：简单易用、能告诉他「今天练什么」、能记录下来看到自己的进步
- **付费意愿**：中等偏低，对订阅制敏感；可接受一次性买断30–50元
- **使用场景**：球馆练球时随手记录，回家后看内容学习

### 2.2 Persona B — 进阶玩家/业余联赛参赛者「老张」

- **年龄**：28–45岁
- **台球经历**：打球5年以上，参加过业余联赛或俱乐部赛事
- **学习方式**：系统看斯诺克教学，有时请职业教练一对一指导
- **核心痛点**：训练数据散乱（记在本子上或根本不记），训练计划靠经验，难以量化进步
- **期望**：精细化训练数据记录（哪类球命中率、母球控制精度）、可查阅历史训练、制定周期计划
- **付费意愿**：高，愿意为专业工具付费；可接受订阅制100–200元/年
- **使用场景**：每日训练前计划，训练中记录，训练后复盘

### 2.3 Persona C — 培训机构教练「王教练」（v2.0 目标用户）

- **身份**：台球培训机构教练，管理10–30名学员
- **核心痛点**：学员训练进度难以统一追踪；课程内容分发依赖微信；无统一记录工具
- **期望**：学员管理、课程/作业分配、学员训练数据汇总查看
- **使用场景**：机构管理后台 + 教练端 App
- **备注**：此用户需要 B 端产品逻辑（多角色、机构账号），**v1.0 暂不覆盖**

## 3. 关键场景与任务

### 场景一：「今天去球馆，练什么？」（Persona A + B）
1. 打开 App 查看今日训练计划（或系统推荐）
2. 按计划完成 drill，随手记录完成情况
3. 练完记下心得/感想

### 场景二：「复盘最近一段时间的训练」（Persona B 为主）
1. 查看过去一周/一月的训练日志
2. 查看各类型 drill 的成功率趋势
3. 发现薄弱项，调整下阶段训练计划

### 场景三：「不知道某个技术怎么做，查资料」（Persona A 为主）
1. 在知识库搜索「加塞」「侧旋」「母球控制」
2. 阅读图文说明，看示意图
3. 收藏或记笔记，下次练球时针对性练习

## 4. 痛点与期望

| 用户 | 核心痛点 | 期望的解决方案 |
|---|---|---|
| Persona A | 内容碎片化、不知练什么 | 结构化知识 + 系统训练计划 |
| Persona A | 看完就忘、无法系统学习 | 收藏/笔记 + 进度打卡 |
| Persona B | 训练数据无结构记录 | 训练日志 + 数据可视化 |
| Persona B | 训练计划靠经验拍脑袋 | 可定制的周期训练计划 |
| Persona C（v2.0） | 学员进度难统一管理 | 机构管理后台 |

## 5. 约束（设备、网络、技能、地区）

- **设备**：iPhone 优先，v1.0 不考虑 iPad 优化（假设，待确认）
- **网络**：球馆网络通常良好，但需考虑基本离线功能（训练记录不应依赖网络）
- **地区**：中国大陆，内容需符合国内合规要求（视频内容资质问题）
- **语言**：简体中文优先

## 6. 待验证假设与验证计划

| 假设 | 验证方式 | 优先级 |
|---|---|---|
| 爱好者愿意为训练工具付费 | 小规模问卷（目标台球爱好者社群）| 高 |
| 进阶玩家有精细化数据记录需求 | 5–10人深度访谈 | 高 |
| AI生成图文教学内容质量可被用户接受 | 内容样本测试 | 中 |
| 用户愿意在练球时使用手机记录 | 观察性研究/访谈 | 中 |
