# 技术架构

## 1. 架构概览

```
┌─────────────────────────────────────────┐
│           iOS App（SwiftUI）             │
│                                         │
│   训练 │ 动作库 │ 角度 │ 历史 │ 我的    │
│                                         │
│   本地层（SwiftData）                    │
│   ├── 训练记录（离线可用，后台同步）      │
│   └── 角度测试历史（离线可用，后台同步）  │
└─────────────────────────────────────────┘
          │                    │
    用户数据层              内容数据层
          │                    │
┌─────────────────┐   ┌────────────────────┐
│   LeanCloud     │   │  CloudKit（公开库） │
│                 │   │                    │
│ 用户认证：       │   │ Drill 内容         │
│  微信登录        │   │ 官方训练计划        │
│  手机验证码      │   │ （只读，热更新）    │
│  Sign in Apple  │   │                    │
│                 │   │                    │
│ 用户私有数据：   │   └────────────────────┘
│  训练记录同步    │
│  角度测试历史    │
│  用户设置偏好    │
└─────────────────┘
```

**核心原则**：
- **离线优先**：训练记录、角度测试、动作库浏览均可在无网络（球馆）下完整使用
- **无自建服务器**：LeanCloud（用户数据）+ CloudKit（内容数据），均为托管服务
- **内容热更新**：Drill 内容存于 CloudKit 公开库，无需发版即可修改/新增
- **中国合规**：LeanCloud 数据节点在中国大陆，CloudKit 通过云上贵州访问

## 3. 用户认证方案（LeanCloud）

### 支持三种登录方式

| 登录方式 | 适用场景 | 实现 |
|---|---|---|
| **微信登录** | 主流，一键登录，用户体验最好 | LeanCloud 微信 OAuth，需注册微信开放平台 |
| **手机验证码** | 托底方案，不用微信的用户 | LeanCloud 短信服务（内置），国内短信通道 |
| **Sign in with Apple** | App Store 合规要求 | LeanCloud Apple 登录支持 |

> **App Store 规则**：提供任何第三方登录时，必须同时提供 Sign in with Apple，已满足。

### 登录流程设计

```
首次打开 App
  → 引导页（可跳过，匿名浏览动作库）
    → 开始训练记录时提示登录
      → 登录选择页
          ├── [微信登录]（推荐，最上方）
          ├── [手机号登录]（输入手机号 → 收验证码 → 登录/注册）
          └── [Apple ID 登录]

匿名用户：
  - 可浏览动作库、使用角度训练（纯本地）
  - 训练记录仅存本地，不同步云端
  - 提示：「登录后数据可多设备同步」
```

### 微信开放平台申请要求

> ⚠️ 需提前准备：
> - 注册微信开放平台账号（open.weixin.qq.com）
> - 提交 App 审核（需要 App Store 已上线或 TestFlight 截图）
> - 配置 AppID + Universal Links（iOS 微信回调）
> - 审核周期通常 1–3 个工作日

## 4. 数据架构

### 4.1 本地数据（SwiftData，离线主存储）

```
TrainingSession（训练记录）
├── id: UUID
├── date: Date
├── ballType: BallType（中式台球 / 9球）
├── duration: Int（分钟）
├── notes: String（心得备注）
├── sets: [DrillSet]
└── syncStatus: SyncStatus（本地 / 已同步 / 待同步）

DrillSet（每个 Drill 的练习记录）
├── drillId: String
└── groups: [DrillGroup]

DrillGroup（每一组）
├── targetBalls: Int（如 15）
└── madeBalls: Int（实际进球数）

AngleTestSession（角度测试历史）
├── id: UUID
├── date: Date
└── questions: [AngleQuestion]

AngleQuestion
├── actualAngle: Int
├── answeredAngle: Int
├── pocketType: PocketType（角袋 / 中袋）
└── isCorrect: Bool（±3° 容差）
```

### 4.2 LeanCloud 私有数据（用户登录后同步）

- 与本地 SwiftData 结构一一对应
- 同步时机：每次保存训练记录后后台异步同步
- 冲突策略：以最新 updatedAt 时间戳为准
- 离线写入：本地先保存，网络恢复后自动补同步

### 4.3 CloudKit 公开数据库（Drill 内容）

```
DrillContent（训练项目，约 72 条）
├── id: String
├── name: String
├── category: DrillCategory（8 大类）
├── subCategory: String
├── ballTypes: [BallType]
├── level: TrainingLevel（L0–L4）
├── diagramData: JSON（球台动画路径数据）
├── videoURL: String?（v1.0 暂空，v1.1 填入）
├── description: String
├── tips: String
├── targetBalls: Int
├── passCriteria: [LevelCriteria]
└── relatedDrillIds: [String]

OfficialPlan（官方训练计划，6 套）
├── id: String
├── name: String
├── targetLevel: TrainingLevel
├── durationWeeks: Int
├── sessionsPerWeek: Int
└── weeks: [PlanWeek]
```

**内容更新流程**（无需发版）：
1. 在 CloudKit Dashboard 修改 DrillContent 记录
2. App 启动时检查内容版本号，有更新则后台拉取
3. 新内容写入本地缓存，离线可用

## 5. 球台动画技术方案

**实现**：SwiftUI Canvas + withAnimation，路径数据存于 CloudKit DrillContent.diagramData

**动画数据格式（JSON）**：

```json
{
  "table": { "width": 1, "height": 2 },
  "balls": [
    { "id": "cue", "x": 0.5, "y": 0.8, "color": "white" },
    { "id": "obj1", "x": 0.3, "y": 0.3, "color": "red" }
  ],
  "paths": [
    { "ballId": "cue",  "phase": 1, "points": [{"x":0.5,"y":0.8}, {"x":0.3,"y":0.35}] },
    { "ballId": "cue",  "phase": 2, "points": [{"x":0.3,"y":0.35}, {"x":0.5,"y":0.6}] },
    { "ballId": "obj1", "phase": 2, "points": [{"x":0.3,"y":0.3}, {"x":0.3,"y":0.05}] }
  ],
  "annotations": [{ "type": "pocket_target", "pocketId": "tm" }]
}
```

**动画阶段**：
1. 静态展示初始球位 + 目标袋口高亮
2. 母球运动路径动画（击球到接触）
3. 母球走位 + 目标球进袋同时播放
4. 循环播放 / 用户可手动暂停

## 6. 角度感知训练技术方案

纯本地计算，零网络依赖：

```swift
// 核心公式
func contactPointOffset(cutAngle: Double) -> Double {
    return sin(cutAngle * .pi / 180)  // 返回 0–1，1 = 球半径
}

// 自适应权重出题（30% 基础 + 70% 误差加权）
func nextQuestion(history: [AngleRecord]) -> AngleQuestion {
    let weights = computeWeights(history: history)   // 各 5° 区间误差加权
    let angle = weightedRandom(weights: weights)      // 按权重抽角度
    let pocketType = choosePocketType()               // 60% 角袋 / 40% 中袋
    return AngleQuestion(angle: angle, pocketType: pocketType)
}

// 专业容差 ±3°
func isCorrect(actual: Int, answered: Int) -> Bool {
    return abs(actual - answered) <= 3
}
```

## 7. 第三方服务汇总

| 服务 | 用途 | 费用 |
|---|---|---|
| **LeanCloud** | 用户认证（微信/短信/Apple）+ 私有数据同步 | 免费额度：3GB 存储，3万次 API/天，够用 |
| **微信开放平台** | 微信 OAuth 登录 | 免费（需企业或个人开发者认证） |
| **Apple CloudKit** | Drill 公开内容库（热更新） | 免费 |
| **Sign in with Apple** | App Store 合规登录 | 免费 |
| **视频 CDN**（v1.1） | 教学视频托管播放 | 届时评估（推荐七牛云/阿里 OSS） |

**刻意不引入**：
- Firebase / 任何 Google SDK（国内访问不稳定）
- 广告 SDK
- 第三方崩溃上报（v1.0 用 Apple 原生 MetricKit，后续按需加）

## 8. 安全与隐私

- **用户训练数据**：存于 LeanCloud，用户私有，需登录才可访问
- **匿名用户数据**：本地 SwiftData 存储，不上传云端
- **微信登录**：仅获取微信 OpenID + 昵称/头像，不获取通讯录/好友
- **短信验证码**：手机号经 LeanCloud 加密存储
- **App Store 隐私清单**：需申报「账号信息（手机号）」「使用数据（训练记录）」
