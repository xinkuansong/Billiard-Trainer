# 设计决策日志 — P2-04

## 决策 1: Apple 标准 Onboarding 模式
- **上下文**: 参考截图（训记）使用极简 splash 风格（品牌名+诗句），需决定球迹的引导页模式
- **选项**: A) 极简 splash 风格（仅品牌名+标语）; B) Apple 标准 feature walkthrough（Logo+功能行+CTA）
- **决策**: 采用方案 B — Apple 标准 Onboarding 模式
- **理由**: 球迹功能较多（训练记录+角度训练+统计），首次启动需明确传达核心价值，Apple Health/Fitness 等原生应用的 feature row 模式更适合功能展示
- **日期**: 2026-04-05

## 决策 2: 功能图标统一品牌绿
- **上下文**: 其他页面（如 ProfileView P2-03）菜单图标使用多色圆形底色，需决定 Onboarding 的图标配色策略
- **决策**: 3 个功能图标统一使用品牌绿 #1A6B3C + 浅绿圆形底色 rgba(26,107,60,0.1)
- **理由**: Onboarding 作为品牌首屏，需要统一、聚焦的视觉调性，多色会分散品牌感，统一绿色强化台球主题
- **日期**: 2026-04-05
