## ⚠️ 观察项（非阻塞，供开发参考）

### 观察 1: P1-09「记录」大标题渲染差异

- P1-09 截图中「记录」大标题呈现为居中偏小的样式，与 P1-07/P1-08 帧1/P1-10 的左对齐大标题视觉略有不同
- 四者共享同一「记录」Tab 根页面，理应大标题样式完全一致
- **原因**: Stitch 各次独立生成的布局微差，非设计决策
- **处置**: 开发时统一使用 iOS native large title（左对齐 34pt Bold Rounded 黑色），以 P1-07 为视觉基准

### 观察 2: P1-09 卡片左侧绿色边线

- P1-09 StatisticsView 的图表卡片有左侧品牌绿竖线装饰，P1-06 AngleHistoryView 无此装饰
- 两者同属数据统计类页面，但装饰风格不完全统一
- **处置**: P1-09 决策 3 已明确此装饰限定于「训练统计页面」，P1-06 为角度测试历史属独立模块。开发时可选择是否统一；如统一，建议两个页面都添加

### 观察 3: Tab 栏图标 Stitch 渲染差异

- 不同 Stitch 生成批次中 Tab 栏的 SF Symbol 图标存在细微差异（如 P1-01 vs P1-07 的图标形状略有不同）
- 这是 Stitch 已知限制，每次生成独立解释 SF Symbol 名称
- **处置**: 开发时统一使用设计文档定义的 SF Symbol 名称（训练: figure.pool.swim / 动作库: book.fill / 角度: angle / 记录: calendar / 我的: person.fill），无需参考 Stitch 渲染的具体图标形状

### 观察 4: P1-01 与 P1-02 右上角图标差异

- P1-01 右上角为搜索+筛选图标，P1-02 为收藏+排序图标
- 两者同属 DrillListView 不同状态，按钮功能应一致
- **原因**: Stitch 在不同版本中对「右上角图标按钮」的解释不同
- **处置**: 开发时统一使用 P1-01 版本的图标配置（搜索 + 筛选），两种状态共享同一导航栏
