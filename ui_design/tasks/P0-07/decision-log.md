# 设计决策日志 — P0-07

## 决策 1: BTAngleTestTable 使用横版 2:1 比例
- **上下文**: v1 中 Stitch 将球台渲染为竖版 1:2（比屏幕还高），不符合真实球台俯视比例
- **选项**: A) 竖版 1:2（屏幕利用率高但不真实）; B) 横版 2:1（符合真实球台比例）
- **决策**: 采用横版 2:1，与 P0-04 底部球台示意和 A-08 BTBilliardTable 组件一致
- **理由**: 角度测试的核心是让用户估算切入角，球台方向必须与真实台面一致，否则角度感知会被误导
- **日期**: 2026-04-04

## 决策 2: 结果反馈包含教育提示文案
- **上下文**: Stitch 在结果卡中自动添加了「提示：容差 ±3° 内为正确。注意观察目标球与底袋的连线，再反推击球点。」
- **决策**: 保留该教育提示，作为结果反馈的一部分
- **理由**: 角度测试是学习工具，提示文案有助于用户改进估算方法，增加 App 的教学价值
- **日期**: 2026-04-04
