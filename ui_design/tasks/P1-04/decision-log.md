# 设计决策日志 — P1-04

## 决策 1: Pro 锁定页底部按钮从品牌绿改为金色
- **上下文**: P1-03 底部操作栏右侧为品牌绿「加入训练」，P1-04 为锁定态需决定底部主操作按钮的样式
- **选项**: A) 保持绿色但文案改为「解锁 Pro」; B) 改为金色 #D4941A 填充「解锁 Pro」
- **决策**: 底部右侧按钮改为金色 #D4941A 填充 + 白字「解锁 Pro」
- **理由**: 金色与页面内的渐进式锁视觉系统一致（锁图标、描边按钮均为金色系），形成统一的 Pro 转化视觉通道。金色填充按钮（底部强 CTA）vs 金色描边按钮（内容区轻引导）形成有意的行动力度差异
- **日期**: 2026-04-04
- **影响范围**: P1-10（StatisticsView Pro 锁定态）可沿用此底部金色 CTA 模式

## 决策 2: 锁定态隐藏而非模糊下方内容区块
- **上下文**: P1-03 有技术维度、视频、训练建议等多个区块，需决定锁定态下这些区块的处理方式
- **选项**: A) 全部模糊化展示（类似统计页的全遮罩）; B) 完全不渲染
- **决策**: 技术维度、视频等区块在锁定态完全不显示，页面在渐进式锁后直接结束
- **理由**: 渐进式锁的核心理念是"展示足够价值以激发付费意愿"，训练要点区已完成此目的。继续模糊化下方区块会增加页面长度但不增加转化价值，反而可能分散用户对解锁按钮的注意力
- **日期**: 2026-04-04

## 决策 3: 锁图标使用圆形琥珀容器增强可见性
- **上下文**: A-04 组件表中锁图标直接放在白色背景上，P1-04 实际页面上下文中需考虑可见性
- **决策**: 接受 Stitch 创意 — 锁图标放在 56pt 浅琥珀圆形容器中（#FFDDAF 底 + 深金图标），增强视觉焦点
- **理由**: 在白色卡片背景上，纯金色图标的视觉重量不足；圆形容器提供了更好的视觉锚点，与操作图标行的圆形按钮形成呼应
- **日期**: 2026-04-04
