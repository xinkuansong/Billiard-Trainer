# 设计决策日志 — A-03

## 决策 1: BTLevelBadge 五级颜色方案
- **上下文**: 需要确定 L0~L4 五个等级的颜色映射，使其在小尺寸胶囊中可辨
- **选项**: A) 全部使用品牌绿的深浅变化; B) 使用多色方案（绿→琥珀→橙→红）表达难度递增
- **决策**: 采用多色方案 — L0/L1 品牌绿、L2 琥珀色、L3 橙色、L4 红色
- **理由**: 多色方案直觉映射难度递增（绿=安全/入门 → 红=高阶/挑战），与交通灯隐喻一致，用户无需阅读文字即可感知等级差异
- **日期**: 2026-04-03

## 决策 2: BTLevelBadge 采用浅底+深色文字风格（L1~L4）
- **上下文**: L0 使用品牌绿实心填充+白色文字，其余等级需要决定填充样式
- **决策**: L1~L4 使用 12%~15% 不透明度的浅色背景 + 深色同色系文字
- **理由**: 浅底风格更轻量，不与 L0 实心填充的"入门推荐"强调效果竞争，同时在 DrillCard 标签行中不过于突兀
- **日期**: 2026-04-03

## 决策 3: BTDrillCard Pro 锁角标位置
- **上下文**: 设计文档要求 Pro 角标在"右上角"，Stitch 生成的位置在卡片右侧与 chevron 上方
- **决策**: 接受当前位置（右侧、chevron 上方），不强制右上角绝对定位
- **理由**: 当前位置在卡片行布局中更自然，且与 chevron 形成垂直对齐，视觉上清晰可辨。严格右上角 overlay 可能遮挡卡片内容
- **日期**: 2026-04-03

## 决策 4: 组件表额外附带等级映射描述表
- **上下文**: Stitch 在 BTLevelBadge 区块中自动添加了等级→描述的映射表（L0 新手基础教学 → L4 职业级对抗演练）
- **决策**: 保留此映射表
- **理由**: 增强了组件文档的实用价值，后续 Phase B+ 的页面设计可以直接参考等级含义
- **日期**: 2026-04-03
