# 设计决策日志 — P2-01

## 决策 1: PlanListView 为 push 子页面模式（非 Tab 根页面）
- **上下文**: PlanListView 可设计为 Tab 根页面或 push 子页面，需确认导航模式
- **决策**: push 子页面模式 — 左侧返回「训练」+ 居中标题「训练计划」+ 右侧「新建」品牌绿文字按钮，无底部 5 Tab 栏
- **理由**: 设计文档 3.1.2 明确 PlanListView 从 TrainingHomeView 进入，是训练 Tab 内的二级页面。导航栏右侧「新建」按钮作为创建自定义计划的唯一入口，简洁清晰
- **日期**: 2026-04-05
- **影响范围**: P2-02 CustomPlanBuilderView 从此页面 push 进入

## 决策 2: 计划卡片沿用 P1-01 缩略图+内容+chevron 布局
- **上下文**: 计划卡片的布局需确认——纯文字 vs 带缩略图
- **决策**: 使用 72pt 台球主题缩略图 + 名称 + 标签行 + 描述 + chevron 的水平布局
- **理由**: C-REVIEW 建议 2 明确推荐沿用 P1-01 确立的列表卡片模式。缩略图增强视觉丰富度和台球主题感，使计划列表浏览体验与动作库一致
- **日期**: 2026-04-05
- **影响范围**: P2-07 FavoriteDrillsView 的列表卡片应沿用同一模式

## 决策 3: Pro 计划详情页顶部 CTA 使用金色填充
- **上下文**: PlanDetailView 显示 Pro 锁定计划时，顶部「开始此计划」按钮应使用品牌绿还是金色
- **决策**: 金色 #D4941A 填充 + 白字 + 锁图标「解锁此计划」
- **理由**: 与 P1-04 底部金色「解锁 Pro」CTA 模式一致（P1-04 决策 1），金色明确传达 Pro 升级信号。顶部金色填充 = 强 CTA，内容区金色描边 = 轻引导，形成有意的行动力度差异（A-04 决策 1 延续）
- **日期**: 2026-04-05
- **影响范围**: 其他需要 Pro 升级入口的详情页可沿用

## 决策 4: PlanDetailView Hero 区使用台球照片
- **上下文**: Stitch 自行在详情页顶部添加了台球场景照片作为 Hero 区
- **选项**: A) 保留照片 Hero; B) 替换为 BTBilliardTable 俯视球台图（与 P1-03 一致）
- **决策**: 保留台球照片 Hero
- **理由**: 训练计划是面向 Drill 集合的概念页面，不像 P1-03 展示单个 Drill 的具体球位需要俯视球台图。照片 Hero 提供了品牌感和视觉吸引力，与 P1-03 的 Hero 模式在结构上一致（顶部全宽视觉区 + 下方信息卡片），仅内容形式不同
- **日期**: 2026-04-05

## 决策 5: DESIGN.md 偏离延续既有模式
- **上下文**: Stitch DESIGN.md 延续 "The Disciplined Athlete" / "The Digital Clubhouse" 主题，primary #005129、No-Line Rule、Glassmorphism 等偏离

- **理由**: 渲染层面品牌色、卡片风格、导航模式均正确
- **日期**: 2026-04-05
