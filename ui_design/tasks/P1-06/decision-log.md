# 设计决策日志 — P1-06

## 决策 1: 统计卡片布局采用 icon+label 水平行 + 大号数值纵向排列
- **上下文**: Prompt 描述为 icon(上) → number(中) → label(下) 垂直堆叠；Stitch 生成为 icon+label 水平行(上) → number(下) 两行布局
- **决策**: 接受 Stitch 的 icon+label 水平行布局
- **理由**: 水平行布局更紧凑且阅读顺序自然（先看图标+标签了解指标类型，再看大号数值），与 Apple Fitness 的统计卡片风格一致
- **日期**: 2026-04-04

## 决策 2: Section 标题使用品牌绿色
- **上下文**: 之前的页面中卡片内 Section 标题通常为黑色；Stitch 在本页面将「误差趋势」和「角度区间分析」渲染为品牌绿 #1A6B3C
- **决策**: 接受绿色 Section 标题
- **理由**: 数据统计页面区块较多，绿色标题有助于快速扫描和区分区块，增强品牌感。此决策局限于数据/统计类页面
- **日期**: 2026-04-04

## 决策 3: 进度条采用细线式（~8px）而非粗条式（24pt）
- **上下文**: Prompt 描述进度条高度为 24pt；Stitch 生成为 h-2（~8px）细线进度条
- **决策**: 接受细线式进度条
- **理由**: 细线式更精致、更现代，与统计页面的简洁风格匹配，避免视觉过重
- **日期**: 2026-04-04

## 决策 4: DESIGN.md 偏离处理延续前序任务模式
- **上下文**: DESIGN.md 使用 primary #005129、No-Line Rule、Glass & Gradient Rule 等偏离项
- **决策**: 延续 A-01 决策 3 — 以 screen.png 和 code.html 为准
- **理由**: 视觉元素通过 primary-container #1A6B3C 正确渲染，不影响最终效果
- **日期**: 2026-04-04
