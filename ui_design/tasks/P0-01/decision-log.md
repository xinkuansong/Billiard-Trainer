# 设计决策日志 — P0-01

## 决策 1: iOS 原生大标题导航栏（非自定义绿色头部）
- **上下文**: Stitch v1 创建了深绿色半透明头部栏（bg-emerald-900/80 + 白色文字），偏离 iOS 原生风格
- **选项**: A) 保留绿色头部栏作为品牌特色; B) 改为 iOS 原生 large title 风格（黑字在默认背景上）
- **决策**: 改为 iOS 原生 large title 风格
- **理由**: 球迹定位为 iOS 原生 App，导航栏应遵循 Apple HIG 的 large title 模式，品牌感通过按钮/卡片/内容区的绿色体现，而非侵入导航栏。这也是后续所有页面的基准导航样式。
- **日期**: 2026-04-03
- **影响范围**: 所有带大标题导航栏的页面（Phase B/C/D）应沿用此风格

## 决策 2: GO! 按钮为圆角矩形（非胶囊）
- **上下文**: Stitch v1 将 GO! 按钮渲染为 pill 胶囊形，需确认按钮形状规范
- **决策**: GO! 按钮使用 rounded-lg（~8pt 圆角矩形），与 A-02 BTButton primary 样式一致
- **理由**: 延续 Phase A 确定的按钮圆角体系（sm=8pt 矩形 / full=胶囊），行动按钮统一为矩形
- **日期**: 2026-04-03

## 决策 3: 筛选 Chip 选中态使用近黑色（非品牌绿）
- **上下文**: 筛选 Chip 行的选中态颜色需要确认——品牌绿还是深色
- **决策**: 选中态使用 #1C1C1E（近黑色）填充 + 白色文字
- **理由**: 筛选 Chip 是辅助导航，非主行动。使用深色而非品牌绿可避免与 BTTogglePillGroup（品牌绿选中态）混淆，也与 iOS 原生筛选 Chip 风格一致。品牌绿色保留给主行动按钮和关键 UI 指示器。
- **日期**: 2026-04-03

## 决策 4: PRO 徽章使用浅底+深色文字风格
- **上下文**: PRO 徽章样式需要确认——实心金色还是浅底风格
- **决策**: 浅金色底 rgba(212,148,26,0.12) + 金色文字 #D4941A + 胶囊形
- **理由**: 与 A-03 BTLevelBadge 的浅底风格一致（L1~L4 使用 12%~15% 不透明度浅底），保持标签系统的视觉统一性
- **日期**: 2026-04-03

## 决策 5: 即将到来区 GO! 按钮禁用态
- **上下文**: Stitch 对「即将到来」区的 GO! 按钮使用了 opacity-50 禁用态，设计文档未明确指定
- **决策**: 接受禁用态处理
- **理由**: 「即将到来」的训练不是今日安排，不可立即开始，禁用态在交互逻辑上合理。后续开发中可根据实际需求调整（如改为"查看"按钮）
- **日期**: 2026-04-03

## 决策 6: DESIGN.md 偏离延续 A-01 模式
- **上下文**: Stitch DESIGN.md 延续 "Precision Tactician" 主题，包含 No-Line Rule、Glassmorphism、primary #005129 等偏离项
- **决策**: 延续 A-01 决策 3 — 以 code.html 和 screen.png 渲染效果为准，DESIGN.md 仅作参考
- **理由**: 一致性处理，渲染层面品牌色和布局均正确
- **日期**: 2026-04-03
