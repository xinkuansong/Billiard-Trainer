# 设计决策日志 — P2-05

## 决策 1: LoginView 为 Sheet 模态而非全屏页面
- **上下文**: LoginView 从 OnboardingView「登录已有账号」或 ProfileView 访客态触发，需确认展示模式
- **选项**: A) 全屏独立页面; B) Sheet 模态弹窗
- **决策**: 采用 Sheet 模态弹窗（拖拽条 + 白色背景 + 圆角顶部）
- **理由**: 设计文档明确 LoginView 从 OnboardingView 和 ProfileView 的 Sheet 弹出，用户可通过下滑关闭返回，降低登录压力
- **日期**: 2026-04-05

## 决策 2: 三按钮视觉层级 — Apple > 微信 > 手机号
- **上下文**: 三种登录方式需要明确的视觉优先级
- **决策**: Apple 登录（黑色实心 #000000）> 微信登录（#07C160 实心）> 手机号登录（白底描边），从深到浅递减
- **理由**: Apple 登录是中国 App Store 合规要求且操作最快（一步完成），微信登录是中国用户最常用方式（一步完成），手机号登录需要多步输入因此最低优先
- **日期**: 2026-04-05
- **影响范围**: E-04 Dark Mode 需适配三种按钮的深色变体
