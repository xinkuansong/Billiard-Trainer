# E-01 Decision Log — Dark Mode 训练流程

## 决策 1: v1 帧1/帧2 直接通过
- **日期**: 2026-04-05
- **背景**: 帧1 (TrainingHomeView) 和帧2 (ActiveTrainingView 训练总览) 的 v1 生成高度忠于 Light Mode 布局
- **决策**: 直接通过，无需修改
- **备注**: 帧1 筛选 Chip 选中态样式略有偏差（白色描边 vs 填充），为已知 Stitch 限制，开发以提示词规范为准

## 决策 2: v1 帧3/帧4 退回重做——强制像素级一致
- **日期**: 2026-04-05
- **背景**: Stitch 将帧3（单项记录）和帧4（训练总结）当作全新设计生成，布局和内容严重偏离 Light Mode
- **问题**: 帧3 丢失 5 列网格改为卡片行，底部错用 Tab 栏；帧4 虚构「平均偏角」「训练强度」等不存在的指标，Drill 名称和数据全部错误
- **决策**: 在同一 Stitch 会话中追加修正指令 + 重新附加 Light Mode 截图，强调 "pixel-identical color remap"
- **结果**: v2 两帧均成功恢复正确布局

## 决策 3: v2 帧3 残留球台边框装饰——不阻塞
- **日期**: 2026-04-05
- **背景**: v2 帧3 球台示意区外围有琥珀色装饰边框，Light Mode (P0-04) 无此元素
- **决策**: 标记为非阻塞观察项，开发时以 P0-04 为准（无边框）

## 决策 4: v2 帧4「详情」标签——不阻塞
- **日期**: 2026-04-05
- **背景**: v2 帧4 训练明细 section header 右侧新增「详情」标签，Light Mode 为「详情」(P0-06 有类似)
- **决策**: 开发时保留或移除均可，跟随 Light Mode 即可

## 决策 5: 帧5a PlanListView 条件通过——Tab 栏为 Stitch 限制
- **日期**: 2026-04-05
- **背景**: 帧5a 底部出现 Tab 栏，但 PlanListView 是 push 子页面不应有 Tab
- **决策**: 条件通过，开发时不显示 Tab 栏。PRO/Level 徽章合并和缺少 chevron 同属开发修正项

## 决策 6: 帧5b PlanDetailView Dark 推迟

- **决策**: 不阻塞 E-01 整体通过。PlanDetailView Dark 作为补充帧，后续生成

## 决策 7: Dark Mode prompt 经验——修正指令模式有效
- **日期**: 2026-04-05
- **背景**: v1 帧3/帧4 证明长提示词中 Stitch 对「pixel-identical」理解有限，v2 使用修正指令 + 重新附加参考图的方式修复
