### 帧3: ActiveTrainingView 单项记录 Dark Mode — ⭐⭐☆☆☆ (2/5) ❌ 需修改

**截图**: `stitch_task_e_01_frame3/screen.png`
**Light Mode 基准**: `tasks/P0-04/stitch_task_p0_04_04/screen.png`

#### ✅ 通过项
- 纯黑背景
- 计时器绿色显示在顶部
- Drill 头部区域（缩略图 + 名称 + 进球统计）基本正确
- 休息设置行 + 模式切换 Chip 存在
- 球台示意区（「球桌演示」）底部可见
- 「+ 添加一组」绿色文字按钮可见

#### ⚠️ 问题项（需修改）

**P3-1（高权重）: BTSetInputGrid 布局完全重构**
- **期望**: 与 P0-04 一致的 5 列表格网格（组号 / 进球 / 总球 / 完成勾选 / 溢出菜单），每行 48pt，已完成行浅绿底 + 勾选圈，当前行绿色边框输入框
- **实际**: Stitch 将网格简化为卡片式行（圆形组号 + 居中分数 + 时间），丢失了 5 列布局结构、已完成行的绿色勾选圈、当前行的绿色边框双输入框
- **影响**: 核心交互区域与 Light Mode 差异过大，无法作为 Dark Mode 色彩映射参考

**P3-2（高权重）: 底部栏错误 — 显示 5 Tab 栏而非自定义工具栏**
- **期望**: 无 Tab 栏。自定义 5 项工具栏（最小化 / 更多 / 添加绿圆按钮 / 心得 / 切换）
- **实际**: 显示标准 5 Tab 栏（训练 / 记录 / 添加 / 统计 / 我的），中间按钮虽为绿色但 Tab 结构错误
- **影响**: 导航模式与 fullScreenCover 模态不匹配

**P3-3（中权重）: 顶部固定头部结构偏差**
- **期望**: 与帧2 一致的双行固定头部（计时器+4图标 / 训练名称+进度），白色毛玻璃 → Dark 深色毛玻璃
- **实际**: 简化为返回箭头 + 居中计时器 + 省略号，添加了「RECORDING SESSION」英文文字。4 个操作图标（闹钟/历史/排序/保存）全部丢失

**P3-4（低权重）: 新增多余元素**
- 左上角出现返回箭头「←」— fullScreenCover 模态不应有标准返回按钮
- 计时器下方出现「RECORDING SESSION」英文标签 — Light Mode 无此元素

**结论**: ❌ **需修改** — BTSetInputGrid 是此页面的核心交互区域，布局重构导致无法作为 Dark Mode 设计参考。建议重新生成时更强调「完全复制 Light Mode 布局」。

### 帧4: TrainingSummaryView Dark Mode — ⭐⭐☆☆☆ (2/5) ❌ 需修改

**截图**: `stitch_task_e_01_frame4/screen.png`
**Light Mode 基准**: `tasks/P0-06/stitch_task_p0_06_trainingsummaryview_02/screen.png`

#### ✅ 通过项
- 纯黑背景
- 导航栏 「< 结束训练」+ 「训练总结」+ 分享图标 ✅
- 底部三级按钮层级（绿色填充「保存训练」→ 绿色描边「生成分享图」→ 灰色文字「查看历史」） ✅ — 与 P0-06 决策 3 一致
- 统计卡片使用 #1C1C1E 背景，无阴影

#### ⚠️ 问题项（需修改）

**P4-1（高权重）: 统计卡片内容与布局完全偏离**
- **期望**: 与 P0-06 一致的 2×3 网格 — 训练时长 48分钟 / 完成项目 3项 / 总组数 12组 / 总进球 87球 / 平均成功率 72%（底行全宽含进度条）
- **实际**: 重设计为 hero 卡片（平均进球率 72% 全宽含进度条）+ 2×2 网格（训练时长 1:24:05 / 总进球数 148 / 平均偏角 1.2° / 训练强度 高）
- **影响**: 「平均偏角」「训练强度」为 Stitch 虚构指标，不在设计文档中。数值也完全不同

**P4-2（高权重）: Drill 明细卡片布局重构**
- **期望**: 球台缩略图 + 名称 + 等级徽章 + 组数信息 + 成功率 + 展开的逐组详情行（第1组 8/10 ✓ ...）
- **实际**: 图标（非球台缩略图）+ 名称 + 简化信息 + 成功率 + 彩色标签（达标/优秀/待加强），**无逐组详情行**
- **影响**: 丢失了训练总结的核心数据——逐组进球明细。Drill 名称也完全不同（「直线球稳定性」等 vs 原「定点红球进袋」等）

**P4-3（中权重）: Tab 栏标签错误**
- **期望**: 训练 / 动作库 / 角度 / 记录 / 我的（5 Tab，P0-01 确立）
- **实际**: 首页 / 发现 / 训练 / 统计 / 我的 — 「发现」「统计」不存在于球迹 App

**P4-4（低权重）: 训练心得区域改名**
- **期望**: 「训练心得」（引用符号 + 标题 + 正文）
- **实际**: 「教练笔记」— 名称偏离

**结论**: ❌ **需修改** — Stitch 将此帧作为全新设计而非色彩映射，导致统计内容、Drill 数据、卡片布局全部偏离 Light Mode 基准。建议重新生成时强制附加 Light Mode 截图并更强调「pixel-identical layout」。
