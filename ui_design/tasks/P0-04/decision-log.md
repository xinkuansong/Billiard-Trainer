# 设计决策日志 — P0-04

## 决策 1: 单项记录页沿用 P0-03 固定头部和底部工具栏
- **上下文**: P0-04 是 P0-03 训练总览中点击 Drill 后展开的详情视图，需决定是否共享相同的页面框架
- **决策**: 完全沿用 P0-03 的双行固定头部（计时器+图标 / 训练名称+进度）和 5 项底部工具栏
- **理由**: 两者属于同一个 ActiveTrainingView 的不同视图模式，共享框架保持用户的空间感知连续性。P0-03 决策 1（白色毛玻璃固定头部）和决策 2（蓝色添加按钮）在此延续
- **日期**: 2026-04-03
- **影响范围**: P0-05 也应沿用此框架

## 决策 2: BTSetInputGrid 热身行使用橙色「热」标记取代组号
- **上下文**: 热身组需要与正式组区分，需决定视觉标记方式
- **决策**: 组号列使用橙色（#F5A623 / tertiary）小矩形标记内含白色「热」字，取代数字组号
- **理由**: 橙色与品牌绿形成明确对比，「热」字简短直观。与 A-07 定义的热身组标记方案一致
- **日期**: 2026-04-03

## 决策 3: 球台示意区放置在网格下方（可折叠）
- **上下文**: 球台示意和组数网格的上下顺序需要确定
- **决策**: 组数网格在上（核心交互），球台示意在下（辅助参考），球台区域可折叠
- **理由**: 逐组记录是此页面的核心任务，应占据最优视觉位置。球台示意是辅助信息，放在下方并支持折叠，用户需要时可展开参考
- **日期**: 2026-04-03

## 决策 4: v4 参考 A-07 后 Tailwind primary 修正为 #1A6B3C
- **上下文**: v1~v3 的 Tailwind config 中 primary 始终为 Stitch 自行选择的 #005129，通过 primary-container #1A6B3C 补偿。v4 附加 A-07 截图参考后，Stitch 将 primary 修正为 #1A6B3C
- **决策**: 采用 v4 作为最终版本，Tailwind primary 正确为 #1A6B3C
- **理由**: 代码层面的品牌色终于与设计规范一致，减少后续开发转换时的混淆
- **日期**: 2026-04-03

## 决策 5: 添加按钮改为品牌绿（覆盖 P0-03 决策 2）
- **上下文**: P0-03 决策 2 确定添加按钮使用 iOS 系统蓝 #007AFF。v4 中 Stitch 将其改为品牌绿 #1A6B3C
- **选项**: A) 维持 P0-03 蓝色决策（蓝色=通用操作 vs 绿色=训练状态）; B) 接受绿色统一品牌调性
- **决策**: 采用品牌绿 #1A6B3C，覆盖 P0-03 决策 2
- **理由**: 统一品牌调性，全页面使用品牌绿作为主色调，减少颜色碎片化。添加按钮通过尺寸突出（56pt 圆形上浮）已足够醒目，不需要额外的蓝色区分
- **日期**: 2026-04-03
- **影响范围**: P0-03 的底部工具栏添加按钮也应回溯更新为品牌绿（如有后续一致性审核）

## 决策 6: DESIGN.md 偏离在 v4 中大幅减少
- **上下文**: v4 的 Tailwind config 从 60+ 颜色精简为 10 个核心色值，primary 正确
- **决策**: v4 代码质量显著优于 v3，作为最终交付版本
- **理由**: 参考 A-07 截图后 Stitch 理解了我们的设计系统，输出更精准
- **日期**: 2026-04-03
