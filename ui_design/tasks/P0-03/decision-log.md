# 设计决策日志 — P0-03

## 决策 1: 全屏训练页使用白色毛玻璃固定头部
- **上下文**: 计时器和信息行作为固定头部，内容在其下滚动。提示词描述计时器在 #F2F2F7 背景上，Stitch 实现为 bg-white/90 + backdrop-blur-md
- **决策**: 接受白色毛玻璃效果
- **理由**: 固定头部 + 毛玻璃模糊是 iOS 运动/训练类 App 的常见模式（如 Apple Fitness+），在内容滚动时提供清晰的视觉分层。后续全屏训练相关页面（P0-04、P0-05）可沿用此模式
- **日期**: 2026-04-03
- **影响范围**: ActiveTrainingView 系列页面（P0-03~P0-05）

## 决策 2: 添加按钮使用 iOS 系统蓝 #007AFF
- **上下文**: 底部工具栏中间的「添加」按钮颜色选择 — 品牌绿还是系统蓝
- **决策**: 使用 iOS 系统蓝 #007AFF
- **理由**: 设计文档原文描述为「大号蓝色圆形按钮」。蓝色与品牌绿形成对比，突出添加操作的独特性。品牌绿色保留给训练状态指示（计时器、进度圆点），蓝色用于通用操作按钮，分工明确
- **日期**: 2026-04-03

## 决策 3: DESIGN.md 偏离延续 A-01~P0-02 模式
- **上下文**: Stitch DESIGN.md 使用 primary #004392、"No-Line Rule" 等创意偏离
- **决策**: 延续 A-01 决策 3 — 以 code.html 和 screen.png 渲染效果为准
- **理由**: 一致性处理，渲染层面品牌色和布局均正确
- **日期**: 2026-04-03
