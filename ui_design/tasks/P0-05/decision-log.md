# 设计决策日志 — P0-05

## 决策 1: 选中角标使用选择顺序（非固定数量）
- **上下文**: DrillPickerSheet 中选中 Drill 的角标数字方案——每张卡片显示固定 "1"（数量）还是 "1, 2, 3..."（选择顺序）
- **选项**: A) 每张卡片角标均为 "1"（表示数量=1）; B) 角标按选择顺序递增 "1, 2, 3..."
- **决策**: 采用选择顺序递增（B 方案），Stitch 的实现即为此模式
- **理由**: 选择顺序让用户一目了然知道选了哪些项以及先后顺序，在需要排序或调整时更直观。固定 "1" 在选择数量较多时缺少信息量
- **日期**: 2026-04-03

## 决策 2: BTRestTimer 卡片底部拖拽条保留
- **上下文**: Stitch 在居中弹层底部添加了 iOS Sheet 风格的拖拽条（handle），通常用于从底部上滑的 sheet 而非居中弹窗
- **决策**: 保留拖拽条
- **理由**: 视觉上提供了一个精致的结束感，且不影响功能。后续开发中可根据实际交互需要移除
- **日期**: 2026-04-03
