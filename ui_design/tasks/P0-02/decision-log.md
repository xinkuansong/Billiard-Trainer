# 设计决策日志 — P0-02

## 决策 1: 空状态无底部固定按钮
- **上下文**: P0-01（有计划态）底部有固定「开始训练」全宽按钮，P0-02 需决定空状态是否保留
- **决策**: 空状态不显示底部固定按钮，CTA 按钮作为 BTEmptyState 组件的一部分居中放置
- **理由**: 空状态下无可开始的训练，固定按钮无意义。将 CTA 集成在空状态组件中更符合 iOS 空状态的标准模式，引导更集中
- **日期**: 2026-04-03

## 决策 2: 「自由记录」按钮使用 Tier 2 纯文字样式
- **上下文**: 需决定第二个 CTA 按钮的样式——secondary（有边框）还是 text-only
- **决策**: 使用 Tier 2 纯文字按钮（无边框无背景，深色文字）
- **理由**: 「选择训练计划」是主路径应最突出（Tier 1 绿色填充），「自由记录」是备选路径，使用纯文字按钮降低视觉权重，引导用户优先选择计划。与 A-02 确立的三级按钮层级体系一致
- **日期**: 2026-04-03

## 决策 3: 空状态图标使用 fitness_center（哑铃）
- **上下文**: BTEmptyState 需要一个 SF Symbol 图标，需选择适合运动训练主题的图标
- **决策**: 使用 fitness_center（哑铃）图标，48pt，品牌绿 30% 不透明度
- **理由**: 哑铃图标直观传达"训练"含义，与训练 Tab 的图标一致（Tab 栏训练图标也是 fitness_center），加深关联性。30% 不透明度使图标柔和不抢眼，符合空状态"引导而非干扰"的设计原则
- **日期**: 2026-04-03

## 决策 4: DESIGN.md 偏离延续 A-01/P0-01 模式

- **理由**: 一致性处理，渲染层面品牌色和布局均正确
