## 帧 1 — TrainingSummaryView（训练数据总结页）

### ✅ 通过项

- **导航栏结构**：左侧「< 结束训练」绿色文字 + 中间「训练总结」粗体标题 + 右侧分享图标，布局与提示词一致
- **统计卡片网格**：2×2 + 1 全宽排列，5 张统计卡（训练时长/完成项目/总组数/总进球/平均成功率），数据正确
- **平均成功率进度条**：72% 填充的品牌绿色进度条，视觉效果良好
- **训练明细区**：3 张 Drill 卡片，包含球台缩略图、名称、等级徽章、组数统计和成功率
- **训练心得区**：引用图标 + 标题 + 内容文字，结构完整
- **底部操作按钮三级层级**：「保存训练」primary → 「生成分享图」secondary → 「查看历史记录」text，层级清晰
- **品牌绿色 #1A6B3C**：Tailwind config 中 primary 已正确设为 `#1A6B3C`，按钮/图标颜色正确
- **无渐变**：所有填充均为纯色

### ⚠️ 问题项（需修改）

1. **【高】Tab 栏标签名称错误**
   - 期望：训练 / 动作库 / 角度 / 记录 / 我的（与 P0-01~P0-05 一致）
   - 当前：训练 / 动作 / 工具 / 历史 / 我的
   - 影响：「工具」≠「角度」、「历史」≠「记录」、「动作」≠「动作库」，与前序任务不一致

2. **【高】页面背景色偏离**
   - 期望：`#F2F2F7`（iOS 系统浅灰）
   - 当前：code.html 中使用 `bg-[#F8F9FA]`（偏白偏冷的 Material 风格灰色）
   - 影响：与 P0-01~P0-05 的 iOS 风格背景不一致

3. **【高】「生成分享图」按钮边框颜色错误**
   - 期望：1.5pt solid `#1A6B3C` 品牌绿边框 + 绿色文字
   - 当前：灰色边框 `#E2E8F0` + 黑色文字 `#1A1C1F`
   - 影响：按钮应是品牌绿色描边的 secondary 层级，当前与设计系统不符

4. **【中】Drill 详情组数行缺失/折叠**
   - 期望：Card 1 展示 4 组各组详情行、Card 2 展示 3 组、Card 3 展示 5 组
   - 当前：Card 1 仅显示第 1 组，Card 2/3 折叠为「各组详情 ▼」
   - 影响：提示词明确要求展开所有组数详情，折叠虽然 UX 合理但偏离规格

5. **【中】辅助文字/标签样式偏离 — Stitch 自行发挥「编辑风」**
   - 期望：标签为 13pt Regular + `rgba(60,60,67,0.6)`（iOS 系统灰）
   - 当前：标签为 11px Bold Uppercase + `#64748B`（Tailwind Slate 蓝灰色）
   - DESIGN.md 中的 "Authority Scale" 和 "Editorial Asymmetry" 概念未在提示词中要求，属于 Stitch 自行创意
   - 影响：与 P0-01~P0-05 的 iOS 原生文字风格不一致

6. **【低】统计卡片 `aspect-square` 导致卡片过高**
   - 当前：4 张小卡片均为正方形比例，内容只占卡片底部一小部分，大量空白
   - 建议：改为自适应高度或约 1.2:1 宽高比，使页面更紧凑

### 💡 建议项（可选优化）

1. **等级徽章应为药丸形**：当前 BTLevelBadge 为圆角矩形（`rounded-md`），A-03 规范要求 corner radius 999pt（药丸形）
2. **Drill 卡片有 1px 边框**：`border border-[#F1F5F9]`，设计系统的 "No-Line" 规则建议仅通过背景色差或阴影区分层级，非强制但建议去除
3. **Drill 2/3 数据与提示词不一致**：Card 2 提示词为「3 组 · 28/30 球 · 93%」，截图为「4 组 · 35/40 球 · 88%」；Card 3 提示词为「5 组 · 28/50 球 · 56%」，截图为「4 组 · 21/40 球 · 53%」。属于样本数据偏差，不影响设计评审
4. **「生成分享图」图标**：提示词要求 `square.and.arrow.up`（分享图标），当前为 `image`（图片图标），建议统一

### ⚠️ 问题项（需修改）

1. **【高】Tailwind config 中 primary 颜色错误**
   - 期望：`primary: "#1A6B3C"`
   - 当前：`primary: "#005129"`（code.html 第 38 行）
   - 提示词中明确标注 **"Do NOT use #005129"**
   - 当前视觉未受影响（因大部分元素用硬编码 `#1A6B3C`），但代码层面有隐患

2. **【中】颜色主题圆圈缺少文字标签**
   - 期望：每个颜色圆圈下方有标签文字（「基础绿」「黑白」「暗夜蓝」「更多」）
   - 当前：只有圆圈，无标签文字
   - 影响：用户无法通过文字识别主题名称

3. **【中】「隐藏球台图」文字被截断**

   - 原因：底部面板宽度不足以容纳三个胶囊
   - 建议：允许换行或减小字号/内边距

### 💡 建议项（可选优化）

1. **字体选择器样式**：当前为分段控件（共享灰色容器内两个按钮），与 BTTogglePillGroup 规范（独立胶囊）略有差异，但功能等价，不强制修改
2. **「黑白」颜色圆圈**：当前用渐变（`bg-gradient-to-tr from-[#1C1C1E] to-[#FFFFFF]`），而提示词描述为「左半黑/右半白」分割，建议改为 CSS 实现的对半分色
3. **分享卡品牌底栏左侧绿色竖条**：提示词要求 3pt 宽的品牌绿左边框条，截图中难以确认是否存在，建议确保保留
