# 设计决策日志 — P0-06

## 决策 1: 统计卡片布局比例
- **上下文**: v1 中 Stitch 将统计卡片设为正方形（aspect-square），导致卡片过高、页面冗长
- **选项**: A) 保持正方形 B) 自适应高度
- **决策**: 自适应高度，内容紧凑排列
- **理由**: 训练总结页内容较多（统计卡片 + Drill 明细 + 心得 + 按钮），紧凑卡片减少滚动量，提升信息密度
- **日期**: 2026-04-04

## 决策 2: Drill 组数详情默认展开
- **上下文**: v1 中 Stitch 将 Drill 卡片的各组详情折叠为「各组详情 ▼」，提示词要求全部展开
- **选项**: A) 折叠（UX 上可减少页面长度）B) 展开（完整展示训练数据）
- **决策**: 全部展开
- **理由**: 训练总结页的核心价值是完整回顾训练数据，用户在此页需要看到每组的进球详情，折叠会增加操作步骤
- **日期**: 2026-04-04

## 决策 3: 底部按钮三级层级样式
- **上下文**: 需要区分「保存训练」（主操作）、「生成分享图」（次操作）、「查看历史」（辅助）三级
- **选项**: A) 全部填充不同颜色 B) 填充 + 描边 + 文字 三级
- **决策**: 品牌绿填充（primary）→ 品牌绿描边（secondary）→ 灰色文字（tertiary）
- **理由**: 与 A-02 确定的 BTButton 三级层级一致，视觉层级清晰且不喧宾夺主
- **日期**: 2026-04-04

## 决策 4: 分享卡成功率颜色编码
- **上下文**: 深色分享卡上需要用颜色区分高/中/低成功率
- **选项**: A) 全部白色 B) 按数值着色
- **决策**: ≥90% 亮绿 `#25A25A` / 70-89% 品牌绿 `#1A6B3C` / <70% 低透明度白 `rgba(255,255,255,0.45)`
- **理由**: 在深色背景上通过颜色饱和度区分成绩等级，高分更醒目，低分弱化避免打击用户积极性
- **日期**: 2026-04-04

## 决策 5: TrainingShareView 无底部 Tab 栏
- **上下文**: 分享卡定制页是从 TrainingSummaryView 导航 push 进入的子页面
- **选项**: A) 显示 Tab 栏 B) 不显示
- **决策**: 不显示底部 Tab 栏
- **理由**: 分享卡定制是沉浸式操作流程，底部已有自定义面板（字体/颜色/选项 + 分享按钮），显示 Tab 栏会与自定义面板冲突且无导航意义
- **日期**: 2026-04-04
