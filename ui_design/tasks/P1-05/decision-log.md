# 设计决策日志 — P1-05

## 决策 1: AngleHomeView 为简洁导航枢纽页
- **上下文**: Stitch v1 自行添加了 Hero 横幅（球台照片 + PRO SESSION）、头像/齿轮图标、统计卡片，将页面变成营销推广页
- **选项**: A) 接受 Stitch 的丰富化设计; B) 回归设计文档定义的简洁入口页
- **决策**: 移除所有非设计规范内容，保持简洁导航枢纽定位
- **理由**: 设计文档 3.3.1 仅定义两张功能入口卡片和历史入口行；统计数据属于 P1-06 AngleHistoryView；页面应保持 3 个清晰入口（角度测试/对照表/历史）的简单结构
- **日期**: 2026-04-04

## 决策 2: 角度训练页无顶部图标按钮
- **上下文**: P0-01 训练 Tab 有右侧图标（朋友+溢出菜单），Stitch 模仿添加了头像+齿轮
- **决策**: 角度训练 Tab 不设顶部图标按钮，大标题独立展示
- **理由**: 角度训练功能模块无社交/设置等辅助入口需求，保持页面简洁；不同 Tab 的导航栏复杂度应由功能需求决定
- **日期**: 2026-04-04

## 决策 3: ContactPointTableView 为纯工具页，无推广内容
- **上下文**: Stitch v1 在对照表底部添加了球台照片推广卡片 "掌握角度，主宰球桌。"
- **决策**: 移除推广卡片，页面在对照表 90° 行后结束
- **理由**: 设计文档 3.3.3 定义此页面为纯参考工具，不含营销元素；教育/工具类子页面应保持功能纯粹
- **日期**: 2026-04-04

## 决策 4: DESIGN.md 偏离处理延续前序任务模式
- **上下文**: 两帧的 DESIGN.md 均使用 primary #005129、No-Line Rule 等偏离项
- **决策**: 延续 A-01 决策 3 — 以 screen.png 和 code.html 为准
- **理由**: 视觉元素通过 primary-container #1A6B3C 正确渲染，不影响最终效果
- **日期**: 2026-04-04
