# 设计决策日志 — A-05

## 决策 1: BTRestTimer 按钮布局为水平并排
- **上下文**: Stitch v1 将 "+30s" 和 "完成" 按钮垂直堆叠，占据过多纵向空间
- **选项**: A) 保留垂直堆叠（移动端常见模式）; B) 改为水平并排（更紧凑）
- **决策**: 采用水平并排布局，"+30s" 在左（secondary）、"完成休息" 在右（primary）
- **理由**: 两个按钮功能权重不同但操作频率相近，并排布局节省空间，让双环进度环在弹层中更突出；"完成休息" 文案比 "完成" 更明确
- **日期**: 2026-04-03

## 决策 2: BTFloatingIndicator 右对齐定位
- **上下文**: Stitch v1 将浮动指示器水平居中，设计文档要求右下角
- **决策**: 强制右对齐（距右边缘 16pt），位于 Tab 栏上方 8pt
- **理由**: 右下角定位符合拇指热区设计原则（用户右手持机时更容易点击），且不遮挡页面主内容区
- **日期**: 2026-04-03
