# 设计决策日志 — P2-03

## 决策 1: ProfileView 为 Tab 根页面，大标题无额外图标
- **上下文**: Stitch 在访客态 v1 自行添加了📍定位图标和⚙齿轮图标在导航栏区域
- **决策**: 移除所有额外图标，大标题区域仅保留「我的」文字
- **理由**: 设计规格中设置入口在菜单列表内（偏好设置行），不需要在导航栏重复。与 P0-01 训练首页、P1-01 动作库等 Tab 根页面模式一致（大标题 + 可选右上角功能图标），ProfileView 不需要右上角按钮
- **日期**: 2026-04-04
- **影响范围**: E-04 Dark Mode 变体需沿用此模式

## 决策 2: 「退出登录」使用 destructive 红色
- **上下文**: Stitch 渲染的退出登录文字颜色偏绿/品牌色
- **决策**: 开发时使用 #C62828（destructive red）
- **理由**: 退出登录是危险操作，应使用 A-02 决策 2 中的 destructive 颜色（Tier 级别之外的独立危险色），与全 App 的 destructive 用色一致
- **日期**: 2026-04-04

## 决策 3: 访客态 Pro 推广卡使用深色背景
- **上下文**: 访客态需要突出 Pro 升级入口
- **决策**: 使用 #1C1C1E 深色背景 + 金色 #D4941A 文字/图标的推广卡
- **理由**: 与参考截图（训记 Pro 推广卡）模式一致，深色卡片在浅色页面上形成强烈视觉对比，吸引用户注意升级入口。P2-06 SubscriptionView 的深色主题也将延续此基调
- **日期**: 2026-04-04
- **影响范围**: P2-06 付费墙深色主题可参考

## 决策 4: 菜单图标采用多色圆形底色方案
- **上下文**: 菜单行左侧图标需要风格确认
- **决策**: 每个图标使用独立颜色的 30pt 圆形底色（红/蓝/绿/金/灰/紫等），SF Symbol 白色
- **理由**: 参考 iOS 系统设置和训记的彩色图标风格，增强菜单行的视觉区分度和功能辨识度。颜色在已登录和访客态保持一致
- **日期**: 2026-04-04
- **影响范围**: P2-07 FavoriteDrillsView 如有菜单行应沿用此风格

## 决策 5: DESIGN.md 偏离延续既有模式
- **上下文**: Stitch DESIGN.md 延续 "The Grandmaster's Ledger" / "The Elite Baize" 主题，No-Line Rule、Glass & Gradient Rule 等偏离
- **决策**: 延续 A-01 决策 3 — 以 screen.png 渲染效果为准，DESIGN.md 仅作参考
- **理由**: 渲染层面品牌色、卡片风格、导航模式均正确
- **日期**: 2026-04-04
