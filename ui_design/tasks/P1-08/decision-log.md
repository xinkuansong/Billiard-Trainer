# 设计决策日志 — P1-08

## 决策 1: 记录 Tab 空状态保留完整日历网格
- **上下文**: 空状态页面是否需要显示完整日历，还是可以隐藏日历只显示 BTEmptyState
- **决策**: 保留完整日历网格（6×7 布局 + 今日标记），日历下方显示 BTEmptyState
- **理由**: 完整日历让用户感知这是一个日历页面，今日绿色圆标记暗示"今天可以开始训练"，与有数据态保持一致的页面框架便于用户心理模型构建
- **日期**: 2026-04-04

## 决策 2: TrainingDetailView 采用 Sheet 模态而非 push 导航
- **上下文**: 从日历训练列表点击进入详情，需要决定导航模式
- **决策**: 采用 Sheet 模态弹窗（拖拽条 + ✕ 关闭 + 内容）
- **理由**: 与设计文档 3.4.1 一致（"点击训练记录 → Sheet 弹出训练详情"），Sheet 模态让用户保持日历页面上下文，方便快速预览后关闭
- **日期**: 2026-04-04

## 决策 3: TrainingDetailView 导航栏保留居中标题
- **上下文**: v2 中导航栏显示居中训练名（✕ + "杆法专项训练" + "存为模版"），与 header 区域的训练名重复
- **决策**: 接受当前布局，导航栏居中标题作为 Sheet 标识
- **理由**: iOS Sheet 模态常见模式是顶部有标题标识当前内容，虽与下方 header 区重复，但提供 Sheet 上下文；开发时可按需精简
- **日期**: 2026-04-04

## 决策 4: BTOverflowMenu 图标在左侧（与 A-06 一致）
- **上下文**: v1 中 Stitch 将图标放在右侧，v2 修正为左侧
- **决策**: 图标在左 + 文字在右，与 A-06 确立的 BTOverflowMenu 布局保持一致
- **理由**: 左侧彩色图标引导视线扫描，右侧文字描述操作，符合从左到右的阅读方向和 A-06 的视觉基准
- **日期**: 2026-04-04
