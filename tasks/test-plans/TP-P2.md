## 五、数据持久化

### 流程 4：收藏数据重启后保持

**步骤**：
1. 收藏 2 个 Drill（记下名称）
2. 从后台彻底关闭 App（上滑清除）
3. 重新启动 App
4. 「我的」→ 收藏夹

**预期**：之前收藏的 2 个 Drill 仍在列表中。

- [ ] 收藏数据在 App 重启后保持

### 流程 5：取消收藏持久化

**步骤**：
1. 进入收藏夹 → 点击其中一个 Drill → 取消收藏
2. 关闭并重启 App
3. 检查收藏夹

**预期**：取消的 Drill 不再出现。

- [ ] 取消收藏在重启后生效

## 六、登录与数据迁移

> 需要有效的 Apple ID。若无法 Sign in with Apple，可跳过此节并记录。

### 流程 6：Sign in with Apple + 迁移提示

**步骤**：
1. 确保 App 中有一些本地数据（至少收藏过 Drill）
2. 「我的」Tab → 「立即登录」→ Sign in with Apple
3. 登录成功后，观察是否弹出「数据同步」Alert

**预期**：
- 登录成功后，ProfileView 切换为已登录状态（显示 Apple ID 信息）
- 弹出 Alert：「检测到本地训练记录，登录后可同步至云端，换机也不会丢失。」
- 按钮：「立即同步」/「暂不同步」

- [ ] 登录成功，ProfileView 显示已登录头部
- [ ] 迁移提示 Alert 弹出
- [ ] 「暂不同步」可关闭 Alert，不触发上传

📸 截图：已登录状态 + 迁移提示 Alert

### 流程 7：退出登录

**步骤**：
1. 已登录状态下，「我的」Tab → 滚动到底部 → 「退出登录」
2. 确认退回未登录状态

**预期**：退出后 ProfileView 恢复未登录引导。

- [ ] 退出登录后恢复 GuestHeaderView
- [ ] 本地收藏数据仍保留

## 七、后端连通性（需联网 + 已登录）

> 若无法完成 Sign in with Apple，此节可由开发者通过日志验证。

### 流程 8：SyncQueue 后台触发

**步骤**：
1. 已登录状态
2. 将 App 切换到后台（按 Home）
3. 等待 2 秒，切换回前台
4. 观察 Xcode 控制台日志

**预期**：看到 `[SyncQueue]` 相关日志，表示 `processQueue` 被触发。

- [ ] scenePhase .active 触发了 processQueue（可从控制台日志确认）

### 流程 9：后端不可达时的容错

**步骤**：
1. 修改 `Config/Debug.xcconfig` 中 `API_BASE_URL` 为一个无效地址（如 `http://192.168.0.999:3000`）
2. 重新构建运行
3. 登录后使用 App

**预期**：
- App 不崩溃
- 后端不可达时，本地功能（浏览 Drill、收藏）正常
- 可能显示「网络连接失败」等提示，但不阻断使用

- [ ] 后端不可达时 App 不崩溃
- [ ] 本地功能（Bundle Drill 浏览、收藏）正常
- [ ] 测试完成后**恢复** `API_BASE_URL` 为正确值
