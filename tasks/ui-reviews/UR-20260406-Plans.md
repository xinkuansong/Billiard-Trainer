### U-01 PlanCard 布局为纵向，设计稿为横向缩略图布局

- **类别**：布局 / 产品规格
- **严重程度**：P1（应该修复）
- **位置**：训练 > 训练计划 > 官方计划卡片
- **现状**：`PlanCard` 使用 `VStack` 纵向布局——等级标签/PRO 标记在上、标题居中、描述和信息芯片在下。无左侧缩略图。
- **预期**：设计稿（P2-01 stitch + E-01 Dark）为横向布局：左侧 72×72 圆角图片缩略图 → 中间文字区（标题、等级+周数、描述）→ 右侧 chevron。参见 `code.html` 第 120–133 行：`flex items-center space-x-4`，图片 `w-[72px] h-[72px] rounded-[10px]`。
- **修复方向**：将 `PlanCard` 改为 `HStack` 布局；左侧添加计划缩略图占位（可用 mini 球台或占位色块），右侧保留 chevron。
- **涉及文件**：`PlanListView.swift` L302–380（`PlanCard`）

### U-05 PlanDetailView premiumGate 使用 fullMask 而非 progressive 锁定

- **类别**：产品规格 / Freemium
- **严重程度**：P1（应该修复）
- **位置**：训练 > 计划详情 > 付费内容门控
- **现状**：`PlanDetailView.swift` L34 使用 `.premiumGate(isPremium:)` 对整个页面施加 `BTPremiumLock.fullMask`，导致未订阅用户看不到任何计划内容（包括标题、统计、第一周安排）。
- **预期**：设计稿（P2-01 stitch）显示：前 1–2 周可见（第 1 周可展开查看训练内容），第 3 周起使用渐变模糊遮罩 + 锁图标 + "后面还有 N 周训练计划" 提示 + 金色解锁 CTA。这是 `BTPremiumLock.progressive(visibleItems:)` 模式。
- **修复方向**：移除页面级 `.premiumGate()`，改为在 `weeksList()` 中对超出可见数量的 week 应用 progressive 模式。免费用户可预览前 N 周（折叠态），后续周隐藏。激活按钮改为金色 "解锁此计划" 样式。
- **涉及文件**：`PlanDetailView.swift` L34, L144–168, `BTPremiumLock.swift`

**总体评价**：三个视图的 Design Token 基础使用良好（间距、字体、颜色大部分正确），但存在几处关键偏差：(1) PlanCard 布局与设计稿差异显著（纵向 vs. 横向缩略图），(2) PlanDetailView 的 premium 门控方式与设计稿意图不符，(3) 多个交互元素触摸目标不足 44pt。P2 级问题多为硬编码值和局部缺失内容。

**建议优先级**：
1. **第一批（P1 核心体验）**：U-05（premium 门控）、U-01（卡片布局）、U-04（统计字体）、U-08/U-09（触摸目标）
2. **第二批（P1 规范一致性）**：U-02（Token 替换）、U-03（BTLevelBadge 复用）、U-07（chevron 补充）
3. **第三批（P2 打磨）**：U-06/U-13/U-15/U-16（Token 化）、U-10/U-11（Section 顺序）、U-12/U-17（内容补充）
