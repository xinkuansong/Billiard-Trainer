### 新发现问题

### N-01 计划列表分组粒度过细 — 每组仅含 1 个计划卡片
- **类别**：产品规格 / 布局
- **严重程度**：P2
- **位置**：训练 > 训练计划 > 官方计划分组结构
- **现状**：`groupedPlans` 按 `targetLevel` 分组，当前数据产生 4 个组（入门计划/初级计划/进阶计划/中级计划），每组仅 1 张卡片。页面呈现为「标题 → 卡片 → 标题 → 卡片 → …」碎片化视觉
- **预期**：设计稿使用 3 个组（入门计划/中级计划/高级计划），每组包含 2+ 张卡片，信息密度更高、分组更有意义
- **修复方向**：调整 `groupedPlans` 分组逻辑，将相近等级合并（如 L0→L1 + L1 归为「入门计划」，L1→L2 + L2 归为「中级计划」，L3 + L3→L4 归为「高级计划」）；或根据计划 JSON 元数据增加显式分组字段
- **路由至**：swiftui-developer / content-engineer
- **代码提示**：`PlanListView.swift` L23-L30（`groupedPlans` 计算属性 + `levelOrder`），L270-L280（`titleForLevel` 映射）

---

### N-02 PRO 徽章位置仍在标签行而非标题行右侧
- **类别**：产品规格 / 布局
- **严重程度**：P2
- **位置**：训练 > 训练计划 > 付费计划卡片、训练 > 计划详情 > Header 区域
- **现状**：PRO 胶囊在 `HStack` 中与 `BTLevelBadge` + "X 周" 同行显示（标签行）。列表卡片和详情页 header 均如此
- **预期**：设计稿中 PRO 徽章紧贴标题文字右侧，与 `h4`（17px semibold）同行：`<h4>走位突破训练</h4> <PRO badge>`。标签行仅含等级徽章 + 周数
- **修复方向**：将 PRO badge 从标签 `HStack` 移至标题 `HStack`。列表卡片中：`HStack { Text(plan.nameZh) + PRO badge }`；详情页中：`HStack { Text(plan.nameZh).font(.btTitle) + PRO badge }`
- **路由至**：swiftui-developer
- **代码提示**：`PlanListView.swift` L340-L368（PlanCard body — 将 L353-L361 的 PRO badge 移至 L341-L344 的标题区域）；`PlanDetailView.swift` L80-L102（planHeader — 将 L87-L95 的 PRO badge 移至与计划名称同行）

### N-03 周卡片内边距偏紧（12pt vs 设计 20/16pt）
- **类别**：Design Token
- **严重程度**：P2
- **位置**：训练 > 计划详情 > 训练安排 > 周卡片
- **现状**：`weekHeader` 使用 `padding(Spacing.md)` = 12pt 统一内边距；展开内容区同为 `Spacing.md`
- **预期**：设计稿 `px-5 py-4`（水平 20px，垂直 16px）用于 weekHeader，内容区 `px-5 pb-5`（水平 20px，底部 20px）
- **修复方向**：weekHeader padding 改为 `.padding(.horizontal, Spacing.xl).padding(.vertical, Spacing.lg)`（20pt / 16pt）；展开内容区改为 `.padding(.horizontal, Spacing.xl).padding(.bottom, Spacing.xl)`
- **路由至**：swiftui-developer
- **代码提示**：`PlanDetailView.swift` L234（weekHeader padding）、L204-L205（展开内容 padding）
