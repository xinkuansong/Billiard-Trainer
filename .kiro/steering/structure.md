# 文档与目录结构说明

## 仓库根目录

| 路径 | 用途 |
|------|------|
| [AGENTS.md](../../AGENTS.md) | 智能体角色、八阶段工作流、文档与调研总规则 |
| `.cursor/rules/` | Cursor 规则：行为 + 文档管理 |
| `.cursor/skills/ios-product-design/` | 产品设计技能：流程、模板、调研指南 |
| `.kiro/steering/` |  steering：产品上下文与本结构说明 |
| `docs/` | 产品文档与讨论记录 |
| `docs/research/` | 联网调研原始记录与摘录 |

## `docs/` 编号约定

| 文件 | 内容 |
|------|------|
| `00-讨论记录.md` | 按日期的决策与待办，可追溯 |
| `01-产品愿景与定位.md` | 愿景、定位、价值主张、边界 |
| `02-用户研究.md` | Persona、场景、假设与验证 |
| `03-竞品分析.md` | 对比矩阵与差异化 |
| `04-功能规划.md` | 用户故事、优先级、MVP 功能集 |
| `05-信息架构与交互设计.md` | IA、流程、关键状态 |
| `06-技术架构.md` | 技术选型、数据、安全与非功能 |
| `07-路线图与MVP.md` | 里程碑与版本节奏 |
| `08-商业化与合规.md` | 收入模式、隐私、审核与合规 |

## 文档状态（建议取值）

- **草稿**：仅有骨架或初步笔记
- **进行中**：正随讨论填充
- **待评审**：待本人或干系人确认
- **已定稿**：当前阶段冻结（重大变更走修订流程）

## 命名与语言

- 产品文档使用 **简体中文**。
- `docs/research/` 内文件建议：`YYYYMMDD-主题-简述.md`。
