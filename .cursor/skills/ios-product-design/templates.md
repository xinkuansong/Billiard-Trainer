# 产品文档章节模板

以下块可复制到对应 `docs/` 文件；元信息格式建议保持一致。

---

## 通用元信息（置于每个 `01`–`08` 文档顶部）

```markdown
> **状态**：草稿 | **版本**：0.1 | **最后更新**：YYYY-MM-DD
```

---

## `docs/00-讨论记录.md` — 单条记录模板

```markdown
### YYYY-MM-DD — 主题简述

- **参与/备注**：（可选）
- **结论与决策**：
  - …
- **仍待确认**：
  - …
- **已更新文档**：`docs/xx-xxx.md`、…
```

## `docs/01-产品愿景与定位.md`

```markdown
## 1. 背景与问题
（用户痛点、市场缺口）

## 2. 产品愿景
（中长期方向，1–3 段）

## 3. 目标用户与定位
（ primary / secondary 用户，与竞品区隔）

## 4. 价值主张
（一句话 + 支撑论点）

## 5. 成功指标
（定性目标 + 可量化指标，若暂无则写「待定义」）

## 6. 明确不做什么
（范围边界，避免需求膨胀）
```
